06Methodology & provenance

  • Task set. Every task titled "Add README" created after 2026-07-14 — 44 runs, one per (model, effort). Newer non-README tasks are excluded.
  • Validity. A README counts only if the newest 07-phase-2-implement invocation's provider equals the task's headline provider (superseded-inclusive). This is provider-id based, so env-model providers (z.ai, Ollama) that set the model without a --model flag attribute correctly. All 44 are VALID.
  • Quality scores are source-verified 0–10 judgments (accuracy, completeness, clarity) against a fixed Redaction System ground-truth dossier, not measurements. Composite = accuracy×5 + completeness×3 + clarity×2. Accuracy anchors: ≤3 = major hallucination (invented framework/tests/PDO-mysqli/features); 7–8 = accurate + legacy mysql_* identified; 9–10 = fully accurate + honest caveats.
  • Ground truth verified against the actual source this session: no .htaccess/RewriteRule exists (aliases resolve via 404.php+RS_ALIAS_REDIRECT, so "mod_rewrite routing" is a fabrication); the History class genuinely writes the history table (calling it an audit log is accurate); 13 of 14 re-run clones ship no .ddev, so "no DDEV" claims are correct and only Opus/max's clone legitimately documents DDEV.
  • Cost metrics. Duration and tokens are summed across every invocation of a step including superseded retries, so a rerun pays for failed attempts. Input is ~98% cache-read on the Claude family, so output tokens + active duration are the portable cross-model metrics; raw input/total counts are not comparable across providers.
  • Per-step attribution. Under ignore_saved_step_clis every step ran the task's headline model, so per-step metrics reflect that model. The 00-model-health-workflow canary (fixed minimax-m3) is excluded.
  • z.ai model label corrected. The z.ai runs are labelled GLM-5.2 · z.ai (1M), verified from every run's stream_log (glm-5.2[1m]) and the provider ANTHROPIC_DEFAULT_SONNET_MODEL/OPUS_MODEL = glm-5.2[1m] env. The z.ai CLI sets its model through those env vars; the stale cli_providers.model = "glm-4.6" text field does not drive the model and is not what ran. Earlier onboarding reports that labelled z.ai "GLM-4.6" are wrong (June runs were glm-5.1[1m], July runs glm-5.2[1m]).
  • Two GLM-5.2 deployments, kept distinct. GLM-5.2 · z.ai (1M) (hosted 1M-context endpoint, swept low→max) and GLM-5.2 · Ollama (glm-5.2:cloud, single run) are the same base model served two ways; they are listed separately because that is the axis you pin on.
  • Provenance. Metrics from the Haive Postgres DB (tasks, task_steps, cli_invocations); README bodies from each task's feature/readme-bench-* git branch in the haive_repos volume; provider/effort resolved from cli_providers (cli_args --model / effort_level).
