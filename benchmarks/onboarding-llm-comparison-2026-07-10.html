06Notable defects

Correctness failures

MiniMax M3 — fabricated a complete rank/CSRF contract: nonexistent GetRank(), SetRank(), CheckSecHash(), rank constants, admins.rank, and hidden form tokens. Its Reports Wizard skill also reverses DB-prefix behavior and uses PHP 7 ?? syntax.
GLM-4.6 — repeatedly reverses the edit-lock timeout behavior even while citing the correct source lines. Its module agent invents a Mods class and file-backed settings; its KB also claims FCKeditor sanitizes HTML although the input helper merely skips sanitization.
Codex — otherwise precise KB quick references invent six function signatures, including incorrect Insert/Update/Delete/lockTables parameters, and its edit-lock article reverses the same timeout bug.

Routing, portability, and review

Global PHP 5 articles from Kimi and later Claude contain PHP 7/7.3-only APIs (unserialize options and array-form session_set_cookie_params) and an overbroad Argon2 recommendation. Portable does not automatically mean version-correct.
Sandbox-path leaks remain: nine references in Kimi’s rank skill and four in MiniMax’s alias skill point to /haive/workdir.
Broken links: MiniMax has 19 broken skill links; GLM-4.6 and Kimi each have four; Codex has three. The deterministic verification step passed them because it checks structure, not every link target.
Final review missed every issue above. All six review steps reported completion from counts/expected paths. Source-level audit remains necessary; a green pipeline is not an accuracy verdict.

07Recommendations

  1. Default / highest-stakes onboarding: Claude Code. Best composite (8.78), no weak local dimension, and the best balance of depth and global routing. Budget 84.5 active CLI minutes and audit the promoted version-specific guidance.
  2. High-throughput default: GLM-5.2. It delivers 8.25 correctness in 31.8 minutes, about 2.5x Claude’s correctness per compute minute. Add an explicit global-promotion pass and verify theme/form contracts.
  3. Best alternative for agent coverage: Codex. Second overall (8.50), best discovered agents, strong bug-level skills, and moderate compute. Fix the KB API-reference signatures before trusting it as a reference.
  4. Use Kimi when global routing matters. It made the broadest portable promotion set and stayed third overall. Its version-scoped global advice and sandbox paths require review.
  5. Do not select on artifact volume. MiniMax produced the most skills and GLM-4.6 produced very long skills, yet both fall into the lower tier because coherent-looking guides reverse or invent contracts.
▸ Methodology & caveats
  • Ground truth. All six models onboarded separate uploads of the same Redaction System source. Parallel audits checked model-authored claims against the real files and line-level behavior; artifact count alone earned no credit.
  • Detection (20%). Audited LLM output from steps 01/02/04, persisted project metadata, and raw CLI text. Deterministic form values were separated from model judgment.
  • Harness metadata. Every final AGENTS.md project block inherits confirmation-form values mariadb 10.11 and apache. Raw LLM detection instead identified MySQL with no version; scores treat the form values as deterministic. GLM-5.2 was separately penalized for copying the DDEV/MariaDB harness claim into LLM-authored KB guidance.
  • Knowledge base (25%). Read all core docs and every topical category; scored project specificity, coverage, citation quality, factual accuracy, and local/global routing.
  • Skills (25%) and agents (15%). Read every SKILL.md and its subskills. Agent scoring includes only entries with accepted.source = "llm"; the 28 shared scan/template agents and mirrored .agents/skills copies are excluded.
  • Global KB (15%). Queried the separate haive_kb_global.global_kb_entries table by source_task_id, across every status, and read complete bodies plus merge lineage. Later archival does not erase the originating run’s decision. DB lineage is authoritative when a step summary disagrees, as Claude’s 3-merged/1-skipped summary does with its four active attributed rows.
  • No-promotion baseline. A run with zero global articles receives 6.0: the June methodology’s neutral “no contamination, but missed reusable knowledge” floor. It is not a six-out-of-ten assessment of nonexistent content.
  • Cost. CLI minutes = sum of cli_invocations.duration_ms across all rows, including superseded retries. Output tokens use the same all-invocation rule. Raw input/total tokens are excluded because providers account for caching differently.
  • Wall and idle. Wall time includes unattended form gates and is contextual only. Idle uses current plus retry-carried idle. Speed/value rankings use active CLI minutes.
  • Completion is not quality. Every run has 25 step rows and no degraded/failing step. Three or four deterministic/conditional steps were skipped. RAG indexed the same 313 code files in every run and is not scored.
  • Score precision. Dimension scores are evidence-calibrated judgments in 0.5-point increments, not a laboratory measurement. Composite decimals come only from the published weights.
